# rac2 노드 생성

### 1. rac1 VM 종료(복제 전)

- 복제 전에 반드시 rac1을 먼저 종료한다.

```bash
[root@rac1 ~]# shutdown -h 0 -f
```

---

### 2. CMD를 관리자 권한으로 열고 VDI 복제 (호스트: Windows)

```bash
# cmd를 관리자 권한으로 열고 rac1 폴더로 이동
D:
cd D:\oraclevm\rac\rac1

# VDI 복제 (폴더를 만들지 않고 상위 폴더에 복제)
VBoxManage clonemedium disk rac1.vdi ..\rac2.vdi
```

#### 폴더 구조 확인

- 완료 후 `D:\oraclevm\rac\` 구조 예시

```
rac\
├── rac1\      ← rac1.vdi
├── rac2\      ← rac2.vdi  (새로 생성됨)
└── shared\    ← 공유 디스크 (CRS, DATA, FRA)
```

### 6. oracle 계정 `.bash_profile` 수정

- 1번 노드와 다른 부분만 변경한다.

```bash
[root@rac1 ~]# su - oracle
[oracle@rac2 ~]$ vi .bash_profile

export ORACLE_SID=rac2          # ← 2번 노드
export ORACLE_HOSTNAME=rac2     # ← 2번 노드
export GRID_SID=+ASM2           # ← 2번 노드 ASM 인스턴스
```

- 나머지(ORACLE_BASE, ORACLE_HOME, ORACLE_UNQNAME, GRID_HOME, alias 등)는 1번 노드와 동일하다.

![**%s/rac1/rac2/g 로 한번에 변경했습니다.**](rac2%20%EB%85%B8%EB%93%9C%20%EC%83%9D%EC%84%B1/image%2012.png)

**%s/rac1/rac2/g 로 한번에 변경했습니다.**

- `+ASM1`도 `+ASM2`로 변경한다.

#### 핵심 포인트

- `vboxmanage clonemedium`은 폴더를 만들지 않으므로 **`mkdir rac2`를 먼저** 실행해야 한다.
- 게이트웨이는 반드시 **`10.0.2.2`** 이다. (VirtualBox NAT는 `.1`이 아니라 `.2`)
- rac2 변경 필수 4가지: **IP(`10.0.2.16`), 호스트명(`rac2`), ORACLE_SID(`rac2`), GRID_SID(`+ASM2`)**
- 공유 디스크(CRS, DATA, FRA)는 두 노드가 **동일한 디스크를 attach** 해야 한다.
- 디렉터리 권한 설정은 GRID 설치 전에 반드시 선행되어야 한다.
