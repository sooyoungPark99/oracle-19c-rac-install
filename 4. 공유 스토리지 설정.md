#### RAC에서 공유 스토리지가 필요한 이유 ?

1. RAC는 여러 노드(인스턴스)가 같은 데이터베이스 파일에 동시에 접근이 필요
2. 노드별 로컬 디스크를 쓰면 데이터가 분리되어 일관성/동기화 문제가 발생할 수 있어, 공용 스토리지로 단일 데이터 집합을 유지
3. ASM/Clusterware가 디스크를 공동으로 관리하게 해 고가용성(노드 장애 시에도 서비스 지속) 및 확장성을 지원

#### 공유 가상 디스크(VDI) 생성

- Windows 명령 프롬프트를 **관리자 권한**으로 실행 후 `shared` 폴더로 이동해 아래 명령으로 디스크를 생성한다.

디스크가 공유 가능하려면 반드시 **고정 크기(Fixed)** 로 생성해야 한다.

<img width="735" height="499" alt="image" src="https://github.com/user-attachments/assets/ccdc3830-91a6-473c-90cd-12f5b320525e" />

```bash
# 호스트(Windows)에서 수행
vboxmanage createmedium --filename CRS1.vdi --size 1024  --format VDI --variant Fixed
vboxmanage createmedium --filename CRS2.vdi --size 1024  --format VDI --variant Fixed
vboxmanage createmedium --filename CRS3.vdi --size 1024  --format VDI --variant Fixed
vboxmanage createmedium --filename DATA.vdi --size 30720 --format VDI --variant Fixed
vboxmanage createmedium --filename FRA.vdi  --size 20480 --format VDI --variant Fixed
```
<img width="937" height="881" alt="image" src="https://github.com/user-attachments/assets/9b06d850-1ba6-46f9-adcd-cc4a044b3c5e" />
→ shared 폴더에 공유 가상 디스크 5개가 생성되었다면 성공 !

#### 생성된 디스크를 삭제하고 싶다면?

```bash
-- 아래 명령문 실행

vboxmanage closemedium disk CRS1.vdi --delete
vboxmanage closemedium disk CRS2.vdi --delete
vboxmanage closemedium disk CRS3.vdi --delete
vboxmanage closemedium disk DATA.vdi --delete
vboxmanage closemedium disk FRA.vdi  --delete
```

### (이슈) `oracleasm init` 에러 발생

#### 원인 요약
Oracle Linux 8.4와 8.9의 기본 커널 차이로 ASMLib 모듈 호환성 문제가 발생할 수 있음

| 버전 | 기본 커널 | ASMlib 모듈 |
| --- | --- | --- |
| **OL 8.4** | RHCK (4.18.0) 또는 UEK 선택 가능 | `kmod-redhat-oracleasm` 작동 O |
| **OL 8.9** | **UEK (5.15.0) 기본** | `kmod-redhat-oracleasm` 작동 X |

**핵심 체크포인트**

- 설치 전 `uname -r`로 커널을 확인한다.
- UEK 환경이면 `kmod-oracleasm` 등 커널에 맞는 패키지를 사용한다.
</aside>

#### ⚠️ 해결 절차

#### 1) 문제 확인

```bash
# 현재 커널 확인
uname -r

# kmod-oracleasm 패키지 설치(UEK)
yum install kmod-oracleasm

# ASMlib 패키지 확인(3개가 조회되어야 함)
rpm -qa | grep oracleasm
```

<img width="930" height="404" alt="image" src="https://github.com/user-attachments/assets/53b61f33-f2ab-456f-b3ed-1704906bbf44" />

```bash
# ASMlib 초기화 시도(실패)
oracleasm init

# 에러: Loading module "oracleasm": failed
```

<img width="608" height="129" alt="image" src="https://github.com/user-attachments/assets/00d6e952-9237-4808-bb4f-ec28bda33336" />

#### 2) 사용 가능한 커널 확인

```bash
grubby --info=ALL | grep -E "^kernel=|^title="
```

#### 3) 기본 커널을 RHCK(4.18.0)로 변경(필요 시)

```bash
# 방법 1: 커널 경로로 직접 설정
grubby --set-default=/boot/vmlinuz-4.18.0-513.5.1.el8_9.x86_64

# 방법 2: 인덱스로 설정(더 확실)
grubby --set-default-index=1

# 설정 확인
grubby --default-kernel
grubby --default-index
```

<img width="925" height="362" alt="image" src="https://github.com/user-attachments/assets/64086a90-3518-4073-9e01-2265b55daa89" />

#### 4) GRUB 설정 재생성

```bash
# BIOS 부팅용 GRUB 재생성
grub2-mkconfig -o /boot/grub2/grub.cfg

# UEFI 부팅용 GRUB 재생성
grub2-mkconfig -o /boot/efi/EFI/redhat/grub.cfg
```

<img width="767" height="287" alt="image" src="https://github.com/user-attachments/assets/b23811a5-34fa-4823-9940-c6142f368cc8" />

#### 5) 재부팅

```bash
reboot
```

#### 6) 재부팅 후 확인 및 ASMlib 초기화

```bash
uname -r
oracleasm init
oracleasm configure -i
oracleasm status
oracleasm configure
```
