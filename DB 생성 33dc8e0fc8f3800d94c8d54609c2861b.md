# DB 생성

- oracle 유저로 접속 후 `dbca`를 실행한다.

```bash
[oracle@rac1 ~]$ dbca
```

#### Create Database 선택

![image.png](DB%20%EC%83%9D%EC%84%B1/image.png)

#### Advanced configuration 선택

![image.png](DB%20%EC%83%9D%EC%84%B1/image%201.png)

#### RAC/템플릿 선택

- 데이터베이스 유형: **Oracle RAC**
- 구성 유형: **관리자 관리(Admin-Managed)**
- 템플릿: **범용(General Purpose)** 또는 **트랜잭션 처리(Transaction Processing)**

![image.png](DB%20%EC%83%9D%EC%84%B1/image%202.png)

#### 노드 선택

- 모든 노드를 선택한다.

![image.png](DB%20%EC%83%9D%EC%84%B1/image%203.png)

#### DB 이름/SID 및 CDB 설정

- 전역 데이터베이스 이름 및 SID: `racdb`
- **Create as Container database(CDB)** 선택

![image.png](DB%20%EC%83%9D%EC%84%B1/image%204.png)

#### 저장소(ASM/OMF) 설정

- Storage type: **ASM**
- DB file location: `+DATA/{DB_UNIQUE_NAME}`
- OMF 사용 체크(기본값)

![image.png](DB%20%EC%83%9D%EC%84%B1/image%205.png)

#### FRA 및 Archivelog 설정

- FRA 사용 체크 + Storage type: **ASM**
- FRA Size: 환경에 맞게 설정
- 초기부터 아카이브 모드로 생성하려면 **Enable Archiving** 체크

![image.png](DB%20%EC%83%9D%EC%84%B1/image%206.png)

#### 보안 옵션

- Database Vault/Label Security는 필요 시 설정한다.
- 본 구성에서는 미설정(체크 해제) 후 진행한다.

![image.png](DB%20%EC%83%9D%EC%84%B1/image%207.png)

#### 메모리 설정

- 자동 공유 메모리 관리(Automatic Shared Memory Management) 선택 시 물리 메모리의 약 40%가 SGA/PGA에 할당된다.

![image.png](DB%20%EC%83%9D%EC%84%B1/image%208.png)

#### (주의) shmmax 미설정 시 조치

- root에서 `/etc/sysctl.conf`의 `kernel.shmmax` 값을 물리 메모리의 절반(예: 4GB → 4294967296)으로 설정한다.

![image.png](DB%20%EC%83%9D%EC%84%B1/image%209.png)

#### 블록 사이즈/프로세스

- 블록 사이즈는 생성 후 변경이 어렵다.
- 프로세스(PROCESSES)는 이후 변경 가능하며, 예시로 500으로 설정한다(기본값 유지도 가능).

![image.png](DB%20%EC%83%9D%EC%84%B1/image%2010.png)

#### 캐릭터셋/언어

- 캐릭터셋은 생성 후 변경이 까다로우므로 신중하게 선택한다.
- 디버깅 편의상 기본 언어를 **영어**로 선택한다.

![image.png](DB%20%EC%83%9D%EC%84%B1/image%2011.png)

#### 접속 모드

- 기본값인 **전용 서버(Dedicated Server)** 모드를 사용한다(이후 변경 가능).

![image.png](DB%20%EC%83%9D%EC%84%B1/image%2012.png)

#### 샘플 스키마

- 필요 시 샘플 스키마를 생성한다(예: 실습/마이그레이션 테스트용).

![26.png](DB%20%EC%83%9D%EC%84%B1/26.png)

#### 관리 옵션

- 리소스 절약 목적이라면 관리 옵션(EM/CVU 등)을 해제하고 진행한다.

![image.png](DB%20%EC%83%9D%EC%84%B1/image%2013.png)

#### 계정 비밀번호

- 관리 계정 비밀번호를 설정한다.
- 실습 환경에서는 편의상 동일 비밀번호(예: `oracle`)로 설정할 수 있다.

![image.png](DB%20%EC%83%9D%EC%84%B1/image%2014.png)

![image.png](DB%20%EC%83%9D%EC%84%B1/image%2015.png)

#### 추가 옵션(파라미터/스토리지)

- **All Initialization Parameters**에서 초기화 파라미터를 설정할 수 있다.

![image.png](DB%20%EC%83%9D%EC%84%B1/image%2016.png)

![image.png](DB%20%EC%83%9D%EC%84%B1/image%2017.png)

- **Customize Storage Locations**에서 컨트롤 파일 옵션/리두 로그 그룹 추가 등을 설정할 수 있다(필요 시).

#### (참고) RAC 리두 로그 구성

- RAC 환경에서는 기본적으로 노드당 리두 로그 그룹이 2개씩 생성된다.
- 예: 1,2번 그룹 → Thread 1 / 3,4번 그룹 → Thread 2

![image.png](DB%20%EC%83%9D%EC%84%B1/image%2019.png)

#### 생성 옵션

- **데이터베이스 생성(Create Database)** 체크
- **생성 스크립트 저장(Save as a database creation script)** 체크 후 진행

![image.png](DB%20%EC%83%9D%EC%84%B1/image%2020.png)

#### 사전 검사 수행

![image.png](DB%20%EC%83%9D%EC%84%B1/image%2021.png)

- DNS 에러는 SCAN IP가 DNS에 등록되지 않은 경우 발생할 수 있으며, 환경에 따라 무시하고 진행할 수 있다.
- 최소 메모리(예: 8GB) 미충족 경고도 실습 환경에서는 무시하고 진행할 수 있다.

```sql
[root@rac1 ~]# crsctl stat res -t
```
