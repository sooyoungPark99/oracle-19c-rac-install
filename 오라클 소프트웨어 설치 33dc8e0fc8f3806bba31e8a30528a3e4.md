### 4. runInstaller 실행 및 설치 진행(rac1, oracle)

- `runInstaller`를 실행한다.

```sql
[oracle@rac1 ~]$ cd $ORACLE_HOME
[oracle@rac1 19c]$ ./runInstaller
```

- 설치 유형은 **소프트웨어만 설치(Install software only)** 를 선택한다.

![image.png](%EC%98%A4%EB%9D%BC%ED%81%B4%20%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4%20%EC%84%A4%EC%B9%98/image%201.png)

- 설치 옵션은 **Oracle Real Application Clusters** 를 선택한다.

![image.png](%EC%98%A4%EB%9D%BC%ED%81%B4%20%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4%20%EC%84%A4%EC%B9%98/image%202.png)

- `rac1`, `rac2` 노드를 모두 선택한다.
- SSH 연결 테스트는 Grid 설치 단계에서 완료했다면 생략해도 된다.

#### 만약 다음 진행 시 **[INS-06006] Passwordless SSH connectivity not set up between the following node(s)** 오류가 발생했다면

oracle 유저의 SSH 무비밀번호 접속이 rac2로 설정되지 않은 것입니다. rac1에서 oracle 유저로 다시 설정합니다.

![image.png](%EC%98%A4%EB%9D%BC%ED%81%B4%20%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4%20%EC%84%A4%EC%B9%98/image%203.png)

**1. SSH 키 설정 스크립트 실행**

```sql
su - oracle
cd $GRID_HOME/oui/prov/resources/scripts
./sshUserSetup.sh -user oracle -hosts "rac1 rac2" -noPromptPassphrase -advanced
```

yes 입력 후 oracle 비밀번호를 입력합니다.

**2. 양방향 확인**

```sql
ssh oracle@rac1 hostname
ssh oracle@rac2 hostname
ssh oracle@rac1-priv hostname
ssh oracle@rac2-priv hostname
```

모두 비밀번호 없이 되면 설치 화면에서 **OK** → **SSH connectivity** 버튼으로 다시 확인하거나 **Next**로 진행합니다.

### 트러블슈팅

소프트웨어 설치 부분에서 겪은 오류들

- ⚠️ PRVF-5311 또는 unknown option -- O 오류
    
    1. 양쪽 노드 모두에서 scp가 정상 바이너리인지 확인한다.
    
    ```sql
    file /usr/bin/scp
    # ELF 64-bit 로 나와야 정상 (Bourne-Again shell script로 나오면 손상)
    ```
    
    2. 정상 노드에서 손상 노드로 복사
    
    ```sql
    cat /usr/bin/scp | ssh root@<손상노드> "cat > /usr/bin/scp"
    ssh root@<손상노드> "chmod 755 /usr/bin/scp"
    ```
    
- ⚠️ SSH가 정상인데 노드 검증이 계속 실패하는 경우 — 단일 노드 설치 우회
    
    > SSH 양방향 접속이 모두 정상이고 scp 바이너리도 정상인데, 설치 프로그램의 내부 검증(PRVF-5311 등)이 계속 실패하는 경우가 있다. 
    이때는 단일 노드로 설치한 후 rac2에 수동으로 등록하는 방식이 가장 확실하다.
    > 
    
    **1. Nodes Selection 화면에서 rac2 체크 해제**
    
    rac1만 체크된 상태로 Next를 눌러 설치를 진행한다.
    
    **2. 설치 완료 후 rac1에서 ORACLE_HOME을 rac2로 동기화**
    
    ```sql
    # rac1에서 root로 실행
    rsync -avz --delete /u01/app/oracle/product/19c/ root@rac2:/u01/app/oracle/product/19c/
    ```
    
    **3. rac2에서 oracle 유저로 attachHome 실행**
    
    ```sql
    su - oracle
    /u01/app/oracle/product/19c/oui/bin/runInstaller -attachHome -noClusterEnabled ORACLE_HOME=/u01/app/oracle/product/19c ORACLE_HOME_NAME=OraDB19Home1
    ```
    
    **4. rac2에서 root로 root.sh 실행**
    
    ```sql
    su -
    /u01/app/oracle/product/19c/root.sh
    ```
    
- ⚠️ [INS-32123] Remote 'AttachHome' failed on nodes: rac2
    
    **원인:** rac2에 Oracle Inventory가 존재하지 않아 Database Home 등록이 실패
    
    ![image.png](%EC%98%A4%EB%9D%BC%ED%81%B4%20%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4%20%EC%84%A4%EC%B9%98/image%2012.png)
    
    ![image.png](%EC%98%A4%EB%9D%BC%ED%81%B4%20%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4%20%EC%84%A4%EC%B9%98/image%2013.png)
    
    **1. rac2에서 root로 전환하여 oraInst.loc 파일 생성**
    
    인벤토리 위치를 시스템에 알려주는 설정 파일이 없기 때문에 생성한다.
    
    ```sql
    su -
    cat > /etc/oraInst.loc <<EOF
    inventory_loc=/u01/app/oraInventory
    inst_group=oinstall
    EOF
    ```
    
    **2. rac2 root 에서 인벤토리 디렉토리 생성 및 권한 설정**
    
    runInstaller가 인벤토리에 접근할 수 있도록 디렉토리와 권한을 설정한다.
    
    ```sql
    mkdir -p /u01/app/oraInventory/locks
    chown -R oracle:oinstall /u01/app/oraInventory
    chmod -R 775 /u01/app/oraInventory
    ```
    
    **3. rac2에서 oracle 유저로 전환하여 attachHome 수동 실행**
    
    설치 프로그램이 자동으로 하지 못한 Database Home 등록을 수동으로 수행한다.
    
    ```sql
    su - oracle
    /u01/app/oracle/product/19c/oui/bin/runInstaller -attachHome -noClusterEnabled ORACLE_HOME=/u01/app/oracle/product/19c ORACLE_HOME_NAME=OraDB19Home1
    ```
    
    **4. 설치 화면에서 Retry 클릭**
    
- ⚠️ [INS-32123] Execution of orainstRoot.sh script failed on nodes: rac2
    
    > **원인:** rac2에 orainstRoot.sh 파일이 없음 (인벤토리를 수동 생성했기 때문)
    > 
    
    **1. rac2에서 root로 root.sh 실행**
    
    Database Home의 root 스크립트를 먼저 실행한다.
    
    ```sql
    su -
    /u01/app/oracle/product/19c/root.sh
    ```
    
    **2. rac1에서 인벤토리 스크립트를 rac2로 복사**
    
    rac2에 없는 orainstRoot.sh를 rac1에서 가져온다.
    
    ```sql
    scp /u01/app/oraInventory/orainstRoot.sh root@rac2:/u01/app/oraInventory/
    ```
    
    **3. rac2에서 orainstRoot.sh 실행**
    
    ```sql
    # rac2에서 root로 실행
    chmod 755 /u01/app/oraInventory/orainstRoot.sh
    /u01/app/oraInventory/orainstRoot.sh
    ```
    
    **4. 설치 화면에서 Retry 클릭**
    
- ⚠️ 설치 완료 후 — remote nodes 등록 실패 메시지 발생 시
    
    > **원인:** rac2의 인벤토리에 Grid/Database Home 정보가 불완전함
    > 
    
    ![image.png](%EC%98%A4%EB%9D%BC%ED%81%B4%20%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4%20%EC%84%A4%EC%B9%98/image%2014.png)
    
    **1. rac1에서 root로 인벤토리 전체를 rac2로 동기화**
    
    rac1의 완전한 인벤토리를 rac2에 덮어써서 동기화한다.
    
    ```sql
    su -
    rsync -avz /u01/app/oraInventory/ root@rac2:/u01/app/oraInventory/
    ```
    
    **2. rac2에서 root로 권한 재설정 및 orainstRoot.sh 실행**
    
    동기화 후 파일 소유권을 다시 맞추고 인벤토리 스크립트를 실행한다.
    
    ```sql
    # rac2에서 root로 실행
    chown -R oracle:oinstall /u01/app/oraInventory
    chmod -R 775 /u01/app/oraInventory
    /u01/app/oraInventory/orainstRoot.sh
    ```
    
    **3. rac2에서 oracle 유저로 등록 확인**
    
    두 Home(Grid, Database)이 모두 등록되어 있는지 확인한다.
    
    ```sql
    su - oracle
    cat /u01/app/oraInventory/ContentsXML/inventory.xml
    ```
    
    **아래 두 줄이 보이면 정상이다.**
    
    ```sql
    OraGI19Home1 → /u01/app/grid/19c
    OraDB19Home1 → /u01/app/oracle/product/19c
    ```
    
    ![image.png](%EC%98%A4%EB%9D%BC%ED%81%B4%20%EC%86%8C%ED%94%84%ED%8A%B8%EC%9B%A8%EC%96%B4%20%EC%84%A4%EC%B9%98/image%2015.png)
    
- ⚠️  root.sh 실행 시 잘못된 경로 참조 오류
    
    > **원인:** rac2의 ORACLE_HOME에 잘못된 경로(`/u01/app/oracle/product/19.0.0/dbhome_1`)를 참조하는 파일들이 남아 있는 경우. 
    인벤토리 수동 생성이나 부분 동기화를 거치면서 이전 잔여 파일이 정리되지 않아 발생한다.
    > 
    - **오류 메시지 예시:**
    
    ```sql
    [root@rac2 ~]# /u01/app/oracle/product/19c/root.sh
    Performing root user operation.
    ...
    Now product-specific root actions will be performed.
    /u01/app/oracle/product/19c/root.sh: line 3: /u01/app/oracle/product/19.0.0/dbhome_1/install/utl/rootmacro.sh: 그런 파일이나 디렉터리가 없습니다
    /u01/app/oracle/product/19c/network/install/sqlnet/setowner.sh: line 3: /u01/app/oracle/product/19.0.0/dbhome_1/install/utl/rootmacro.sh: 그런 파일이나 디렉터리가 없습니다
    /u01/app/oracle/product/19c/rdbms/install/rootadd_filemap.sh: line 8: /u01/app/oracle/product/19.0.0/dbhome_1/install/utl/rootmacro.sh: 그런 파일이나 디렉터리가 없습니다
    ```
    
    **1. rac2의 root.sh 내용 확인**
    
    스크립트가 잘못된 경로를 가리키고 있는지 먼저 확인한다.
    
    ```sql
    [root@rac2 ~]# head -5 /u01/app/oracle/product/19c/root.sh
    ```
    
    정상이라면 `/u01/app/oracle/product/19c/install/utl/rootmacro.sh` 경로가 보여야 한다. 
    만약 `/u01/app/oracle/product/19.0.0/dbhome_1/...` 경로가 보이면 잘못된 상태이다.
    
    **2. rac1에서 ORACLE_HOME 전체를 `--delete` 옵션과 함께 동기화**
    
    - `-delete` 옵션이 핵심이다. 이전에 잘못 들어간 파일들을 제거하면서 동기화해야 한다.
    
    ```sql
    # rac1에서 root로 실행
    [root@rac1 ~]# rsync -avz --delete /u01/app/oracle/product/19c/ root@rac2:/u01/app/oracle/product/19c/
    ```
    
    **3. rac2에서 root.sh 내용 재확인**
    
    ```sql
    [root@rac2 ~]# head -5 /u01/app/oracle/product/19c/root.sh
    ```
    
    `/u01/app/oracle/product/19c/install/utl/rootmacro.sh` 경로가 보이면 정상
    
    **4. rac2에서 root.sh 재실행**
    
    ```sql
    [root@rac2 ~]# chmod 755 /u01/app/oracle/product/19c/root.sh
    [root@rac2 ~]# /u01/app/oracle/product/19c/root.sh
    ```
    
    **5. 정상 완료 확인**
    
    다음과 같이 오류 없이 끝나면 성공이다.
    
    ```sql
    Performing root user operation.
    The following environment variables are set as:
        ORACLE_OWNER= oracle
        ORACLE_HOME=  /u01/app/oracle/product/19c
       Copying dbhome to /usr/local/bin ...
       Copying oraenv to /usr/local/bin ...
       Copying coraenv to /usr/local/bin ...
    Entries will be added to the /etc/oratab file as needed by
    Database Configuration Assistant when a database is created
    Finished running generic part of root script.
    Now product-specific root actions will be performed.
    ```
