### 5. dnsmasq 설정(로컬 DNS)

#### dnsmasq 설정 변경

`/etc/dnsmasq.conf`에서 아래 항목을 찾아 수정한다.

- 주석(`#`) 제거
- `localnet` → `localdomain`

**수정 전:** `#local=/localnet/`

**수정 후:** `local=/localdomain/`

```bash
vi /etc/dnsmasq.conf
```

<img width="845" height="442" alt="image" src="https://github.com/user-attachments/assets/f9cb4293-519b-46af-8215-34933f8cdc67" />

<aside>

이 설정은 `.localdomain`으로 끝나는 도메인(예: `rac1.localdomain`, `rac-scan.localdomain`) 요청이 오면, 외부 DNS에 묻지 않고 **/etc/hosts**를 참조해 응답하도록 하는 설정이다.

</aside>

#### dnsmasq 기동 및 확인

### 6. /dev/shm(tmpfs) 7GB 할당

Oracle SGA 등을 위해 공유 메모리(`/dev/shm`) 용량을 7GB로 고정한다.

#### /etc/fstab에 항목 추가

```bash
vi /etc/fstab

# 아래 한 줄 추가
tmpfs   /dev/shm   tmpfs   size=7g   0 0

cat /etc/fstab
```

<img width="928" height="402" alt="image" src="https://github.com/user-attachments/assets/e788c58c-e1d1-4ab4-b865-d4b0f38f02b3" />

#### remount 및 반영 확인

```bash
mount -o remount /dev/shm

# systemd가 이전 fstab를 잡고 있으면 reload 필요
systemctl daemon-reload

df -h /dev/shm
```

<aside>

/dev/shm size=7g   →  (이번 설정)

SGA_TARGET=7G    →  (Oracle 파라미터 예정)

→ RAC 인스턴스 정상 기동 목적

</aside>

### 7. (트러블슈팅) ping 8.8.8.8이 안 될 때

### 원인

- 설치 시 게이트웨이를 `10.0.2.1`로 입력했지만, VirtualBox NAT 네트워크의 실제 게이트웨이가 `10.0.2.2`인 경우 발생

### 예방(설치 시 네트워크 설정)

| 항목 | rac1 | rac2 |
| --- | --- | --- |
| IP 주소 | `10.0.2.15` | `10.0.2.16` |
| 서브넷 마스크 | `255.255.255.0` | `255.255.255.0` |
| 게이트웨이 | `10.0.2.2` | `10.0.2.2` |
| DNS | `168.126.63.1` | `168.126.63.1` |

### 이미 설치했는데 ping이 안 될 때(해결)

1. VirtualBox에서 **어댑터1**을 `NAT 네트워크` → `NAT(단순)`로 변경한다. (변경 전 VM 종료)

```bash
poweroff
```

1. VM 부팅 후 게이트웨이를 수정한다.

```bash
nmcli con mod enp0s3 ipv4.gateway 10.0.2.2
nmcli con up enp0s3
```

1. 확인

```bash
ping -c 4 8.8.8.8
```

### Q. NAT(단순)으로 바꾸면 rac1↔rac2 통신이 안 되는지?

- rac1↔rac2 통신은 **어댑터2(호스트 전용, 192.168.56.x)** 로 수행할 수 있다.

| 어댑터 | 방식 | 용도 |
| --- | --- | --- |
| 어댑터1 (enp0s3) | NAT(단순) | 인터넷(yum) 전용 |
| 어댑터2 (enp0s8) | 호스트 전용 | rac1↔rac2 통신 + PuTTY 접속 |

### 4. 공유 메모리(shmmax, shmall) 값 설정

- `/etc/sysctl.conf`의 `shmmax`, `shmall` 값을 메모리의 절반(4GB = 4294967296 바이트)로 설정한 뒤 반영한다.

```bash
[root@rac1 ~]# vi /etc/sysctl.conf
kernel.shmall=4294967296
kernel.shmmax=4294967296
```

### 6. oracle 사용자 `.bash_profile` 수정

- root 사용자에서 oracle 사용자의 `.bash_profile`을 수정한다. (또는 oracle 사용자로 접속해 수정)

```bash
[root@rac1 ~]# vi ~oracle/.bash_profile

# 아래 내용 추가
export ORACLE_BASE=/u01/app/oracle;
export ORACLE_HOME=$ORACLE_BASE/product/19c;
export ORACLE_SID=rac1;        # 2번 노드: rac2
export ORACLE_HOSTNAME=rac1;   # 2번 노드: rac2
export ORACLE_UNQNAME=racdb;
export GRID_HOME=/u01/app/grid/19c;
export GRID_SID=+ASM1;         # 2번 노드: ASM2
export PATH=$ORACLE_HOME/bin:$GRID_HOME/bin:$PATH
export LD_LIBRARY_PATH=$ORACLE_HOME/lib:/lib:/usr/lib;
export CLASSPATH=$ORACLE_HOME/jlib:$ORACLE_HOME/rdbms/jlib;
export EDITOR=vi;

alias grid='export ORACLE_HOME=$GRID_HOME; export ORACLE_SID=$GRID_SID; export PATH=$ORACLE_HOME/bin:$GRID_HOME/bin:$PATH; echo $ORACLE_SID; echo $ORACLE_HOME'
alias db='. ~oracle/.bash_profile;export PATH=$ORACLE_HOME/bin:$GRID_HOME/bin:$PATH; echo $ORACLE_SID;echo $ORACLE_HOME'
alias oh='cd $ORACLE_HOME;pwd'
alias ss='sqlplus / as sysdba'

[root@rac1 ~]# source ~oracle/.bash_profile
```
