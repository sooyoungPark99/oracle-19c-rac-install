#### 설치 목적지

<img width="1282" height="867" alt="image" src="https://github.com/user-attachments/assets/fc5c3f29-059e-47be-b94d-9db2bc047b41" />

- OS 디스크와 Oracle 데이터 디스크(ASM용, FRA용 등)는 역할이 달라진다.
- 실습 환경에서도 "OS 디스크"와 "추가 디스크"를 구분해 이름/용량을 같이 적어두면 이후 RAC/ASM 단계에서 혼동이 줄어든다.

#### 사용자 정의 파티션

사용자 정의를 누른 상태에서 완료를 누르면 다음 화면이 나온다.

<img width="1282" height="867" alt="image" src="https://github.com/user-attachments/assets/0fa5c29b-cb63-4005-9784-a1fc682b0104" />

#### /boot (500MiB)

<img width="1080" height="790" alt="image" src="https://github.com/user-attachments/assets/d81adf1d-e552-4fa3-827b-cd35dd86871f" />

<img width="1280" height="865" alt="image" src="https://github.com/user-attachments/assets/de10cf34-8d7c-469f-9e21-f6a41bd64dfa" />

- /boot는 커널/부트로더 관련 파일이 들어가는 영역이다.
- 커널 업데이트가 여러 번 누적되면 용량이 부족해질 수 있으니, 운영에서는 기준 용량을 더 크게 잡기도 한다.

#### swap (8192MiB)

<img width="1276" height="868" alt="image" src="https://github.com/user-attachments/assets/5b8b3efb-50be-4eef-afc4-3b4e46bfed41" />

- Oracle 설치 사전 점검에서 swap 관련 권고사항이 걸리는 경우가 많다.
- 노드별 RAM이 달라지면 swap 기준도 같이 달라지니, "왜 8GB인지" 기준을 남겨두면 나중에 다시 구성할 때 편하다.

### 5. 네트워크 설정

호스트 이름을 rac1으로 설정하고, enp0s3, enp0s8 둘 다 켠다.

<img width="1266" height="799" alt="image" src="https://github.com/user-attachments/assets/b72f831f-34c9-4884-aa19-5d04f4851046" />

#### DNS 서버 주소 확인

먼저 Windows 명령 프롬프트를 열고 `ipconfig /all`로 DNS 서버 주소를 확인한다.

<img width="870" height="878" alt="image" src="https://github.com/user-attachments/assets/ca496d07-6173-42eb-afb4-f416c59a41b6" />

#### Public IP 설정(enp0s3)

설정으로 들어가 enp0s3(VirtualBox의 어댑터1)을 다음과 같이 public IP로 설정한다.

<img width="1029" height="799" alt="image" src="https://github.com/user-attachments/assets/89bfd5b9-cf85-4891-87c7-44299b1a16b1" />

IPv4를 수동으로 설정한다. 서브넷 마스크 255.255.255.0은 24로 입력해도 같은 값이다.

DNS servers는 위에서 확인한 주소로 입력한다.

<img width="679" height="628" alt="image" src="https://github.com/user-attachments/assets/ffe0b8d9-8136-4143-90b7-5e825040b77e" />
